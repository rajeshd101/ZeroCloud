# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in ZeroCloud, please report it responsibly.

### How to Report

1. **Email**: Send details to [drajesh@hotmail.com](mailto:drajesh@hotmail.com)
2. **Subject**: Include "SECURITY" in the subject line
3. **Details**: Provide as much information as possible:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: We'll acknowledge receipt within 48 hours
- **Investigation**: We'll investigate and assess the vulnerability
- **Updates**: We'll provide regular updates on our progress
- **Resolution**: We'll work to resolve the issue promptly
- **Credit**: We'll credit you in the security advisory (if desired)

### Security Measures

ZeroCloud implements several security measures:

- **End-to-end encryption** using AES-256
- **Local network only** - no internet communication
- **Secure key exchange** for each session
- **No persistent storage** of sensitive data
- **Network isolation** from external networks

### Responsible Disclosure

Please do not:
- Publicly disclose the vulnerability before we've had a chance to fix it
- Access or modify data that doesn't belong to you
- Perform actions that could harm the service or other users

We appreciate your help in keeping ZeroCloud secure!
